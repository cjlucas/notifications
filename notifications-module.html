Module notifications

Client for the iOS app "Notifications" (aka "Push 4.0").

http://www.appnotifications.com/

This module helps scripts to use the HTTP REST API of the `Notifications' application, which is available for the iPhone and the iPod Touch. It supports finding the user's credentials token, and then sending notification in a synchronous or asynchronous way. This modules provides three functions:

get_credentials() -- Get the user's credentials token send() -- Send a notification, waiting for it to be sent. send_async() -- Send a notification, returning immediately, without waiting for the message to be sent.

Version: 0.3.2

Author: Thomas Jost <thomas.jost@gmail.com>, Chris Lucas <cjlucas07@gmail.com>

get_credentials(email, password, debug=False)

 

Get the user's credentials token.

Parameters:
  • email (str) - user's email address
  • password (str) - user's password
  • debug (bool) - writes the response from the API to sys.stderr
Returns: str
A unique token required to access the API, or -1 if there was an error sending the request.
Raises:
  • AssertionError - Is raised if:
    • The HTTP response code is not 200
    • API returned error (usually invalid email/password)
  • ValueError - Is raised if:
    • JSON couldn't be parsed
    • Invalid username/password given
    • Unrecognizable response is received

Send a notification, waiting for the message to be sent.

Parameters:
  • credentials (bytes or str) - the API token returned by get_credentials
  • message - The message used for the alert window when the notification appears.
  • title (str) - The title of the notification in the notifications list. (optional)
  • subtitle (str) - The subtitle of the notification in the notifications list. We suggest you use this to give more informations, for example â€œRSS: a feedâ€. (optional)
  • long_message (str) - The notification when the user select it in the list, this is the long text. (optional)
  • long_message_preview (str) - The preview under the title in the listing, we suggest you donâ€™t put any HTML there. (optional)
  • icon_url (str) - The icon on the left in the listing. (optional)
  • message_level (int) - the importance of the notification, from -2 to 2. (optional)
  • silent (bool) - Should this be silent? (no alert window, just a badge number).
  • action_loc_key (str) - The name of the validation button (optional)
  • run_command (str) - todo
  • sound (int) - Select the sound you want for the notification. The sound should be a number that correlates to this list. (Valid range: 1-40)
  • debug (bool) - writes the response from the API to sys.stderr
Returns: int
The ID of the notification, or -1 if there was an error sending the request.
Raises:
  • ValueError - Is raised if:
    • Invalid user credentials were given
    • Invalid message was given
    • JSON couldn't be parsed
    • sound param is not within valid range
    • message_level is not within valid range

Note: The first two arguments (credentials and message) are mandatory, all of the others are optional. They are the same as the various identifiers described in the documentation of the Notifications HTTP REST API (http://developer.appnotifications.com/p/user_notifications.html).
